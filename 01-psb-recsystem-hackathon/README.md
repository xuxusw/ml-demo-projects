# Рекомендательная система банковских продуктов ПСБ

## О проекте

Рекомендательная система предназначена для персонализированного подбора банковских продуктов ПСБ на основе транзакционной активности клиентов. Система использует поведенческие признаки пользователей, автоматически формирует обучающие данные с помощью LLM и выполняет поиск релевантных предложений.

Проект выполнялся в команде из трех человек.  
**Моя роль:** Data Engineer

## Мои задачи в проекте:
1. Разработка SQL-пайплайна - создание запросов для извлечения и агрегации данных из ClickHouse (транзакции, категории, временные ряды).
2. Построение пользовательских профилей: финансовые показатели, поведенческие паттерны, категориальные признаки.
3. Подготовка обучающей выборки - отбор активных пользователей с достаточной историей.
4. LLM-разметка - автоматическая генерация рекомендаций через Qwen для создания обучающих данных.
5. Формирование витрин данных - подготовка структурированных профилей для модели рекомендаций и FAISS-поиска.

## Clickhouse utils
`load_schema.py` - ETL-пайплайн загрузки данных из Parquet-файлов в ClickHouse.  

`schema.sql` - структура таблиц ClickHouse для хранения пользователей, транзакций, чеков и банковских продуктов.

## Data preparation
`llm_training_users.sql` - формирование выборки пользователей. Фильтрует только активных клиентов, убирает пользователей без истории, оставляет пользователей с достаточным количеством операций (>15).  

`llm_user_features_data.sql` - определение признаков:  
* сумма покупок;
* число транзакций;
* категории покупок;
* подкатегории;
* бренды;
* регион;
* социально-демографический кластер.  

`augment.py` - разметка пользователей через LLM:
* загружает feature table;
* загружает каталог банковских продуктов;
* формирует промпт;
* отправляет его в Qwen;
* получает рекомендации;
* сохраняет результат.  

`products.csv` - каталог продуктов (структура представлена ниже), который LLM использует для формирования рекомендаций.

## Структура `products.csv`
| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `product_id` | `int` | Уникальный идентификатор продукта в системе |
| `product_type` | `str` | Категория продукта (например, "Кредиты", "Карты", "Вклады") |
| `product_name` | `str` | Название продукта, которое показывается клиенту |
| `description` | `str` | Описание продукта - что, для кого, условия |

## Recommendation
`normalized_faiss.py` - подготовка векторных представлений и интеграция поиска похожих пользователей с помощью FAISS.
